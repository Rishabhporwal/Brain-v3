### Judgment calls (deviations from the idealized doc delta, made deliberately)

- **D1 — `first-party-data` kept whole** as `data-platform/first-party-data/` rather than renamed to `data-platform/tracking/`. It contains `identity-resolution`, `reconciliation`, `event-processing`, `schema-registry-svc` — broader than "tracking." The SDK-tracking lane lives at `data-platform/first-party-data/tracking/`. (Doc updated to match.)
- **D2 — `shared-platform/` dissolved, not folded into `shared/`.** `shared/` is for **libraries**; `shared-platform/*` are **services** (skeletons). Folding services into a libs root is a category error. Relocated to owning platforms per CODEOWNERS:
  - `llm-gateway` → `ai-platform/llm-gateway` (ai-platform-owned; placed outside `services/*`/`agents/*` so the uv glob doesn't treat the skeleton as a Python package)
  - `region-adapter` → `platform/region-adapter` (platform-core)
  - `notification-svc` → `platform/notification-svc` (platform-core)
  - `search-svc` → `platform/search-svc` (platform-core)
  - `aggregation-zone` → `data-platform/aggregation-zone` (data concern)
- **D3 — `data-platform/{batch,replay,lakehouse,contracts,streaming}` left in place.** The doc's "reorganize batch/replay into streaming/" is cosmetic on skeletons; skipped to avoid churn.
- **D4 — `data/` is a new top-level physical-data layer** holding `stores/` (Postgres DDL/migrations/seeds) and `warehouse/` (ClickHouse models). This is the load-bearing W1 fix.

## 4. Part 2 — Canonical NestJS service template

The scaffold **already** emits a layered structure. Canonicalize it to the approved 8-folder spec and apply to the BFF:

```
src/
├── api/            # inbound adapters: controllers (http), consumers (kafka), guards
├── application/    # use-cases: commands, queries, ports (interfaces), dto
├── domain/         # entities, value-objects, domain events, domain services (no framework imports)
├── infrastructure/ # outbound adapters: clients, messaging, external integrations
├── persistence/    # repositories (DB adapters) — pulled out of infrastructure per Clean Arch
├── contracts/      # service-local API/event contracts mirroring /contracts (codegen target)
├── config/         # env schema, DI module wiring, configuration
└── main.ts         # composition root (bootstrap)
```

Dependency rule: `api → application → domain ← infrastructure/persistence`. `domain/` imports nothing framework-specific.
